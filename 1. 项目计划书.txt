## 一、项目概述

### 1. 项目背景
在数字化金融时代，银行核心系统的稳定性和数据一致性是金融安全的基石。本项目旨在通过模拟真实银行环境中的核心业务——如开户、资金流转、流水审计等，深入实践软件工程的全生命周期管理。通过该系统的开发，团队将重点攻克高并发下的数据事务处理、前后端分离架构设计以及基于 Git 的规范化协同开发。

### 2. 项目目标
构建一个具备基础银行柜面与用户自服务功能的 Web 系统。实现用户从身份注册到多账户管理、再到实时跨行/行内转账的完整闭环，确保每一笔交易逻辑严密、记录可追溯，并达到数据库事务的 ACID 标准。

### 3. 开发环境与条件
- 操作系统：Windows 10/11, macOS, Linux  
- 后端技术：Python (FastAPI / Flask) + SQLAlchemy (ORM)  
- 前端技术：Vue.js + Element UI + Axios  
- 数据库：MySQL 8.0  
- 版本控制：Git (GitHub 托管)  
- 运行环境：Docker (可选) / Python 虚拟环境

## 二、需求分析

### 1. 目标用户与使用场景
- 普通储户：在线注册并实名后，管理个人多张银行卡，进行余额查询、模拟存取款及向他人转账。  
- 银行管理员：负责监控系统全局交易量、审核新开户请求、对异常账户（如涉诈、挂失）进行冻结操作。

### 2. 核心功能清单
- 用户与鉴权：基于 JWT 的登录态维护、密码加密存储、身份证号唯一性校验  
- 账户管理：个人多账户开立、余额实时统计、账户状态切换（正常/挂失/冻结）  
- 交易中心：  
  - 存取款：模拟现金流入/流出  
  - 转账：支持根据银行卡号进行实时转账，包含余额校验、事务回滚  
- 审计日志：每一笔资金变动均生成唯一的交易流水号，支持分类检索

### 3. 非功能需求
- 性能要求：核心交易逻辑响应时间应在 2 秒以内  
- 安全要求：后端接口需进行身份校验，防止越权查询他人余额  
- 兼容性要求：支持主流浏览器（Chrome、Edge、Firefox）及不同分辨率设备

## 三、技术路线

### 1. 总体架构
采用前后端分离的 RESTful 架构。  
- 前端：负责数据展示与用户交互，通过 Axios 调用后端接口  
- 后端：处理业务逻辑，通过逻辑校验后操作数据库  
- 数据层：MySQL 负责持久化存储，利用数据库事务保证账务安全

### 2. 主要技术
- FastAPI：利用其高性能异步特性处理并发交易请求  
- MySQL 事务：采用 START TRANSACTION 机制确保转账逻辑的完整性  
- Vue.js：动态渲染个人资产概览与动态交易列表

### 3. 数据设计初步考虑
系统初步规划以下核心实体：  
- Users（用户）：存储基本身份信息  
- Accounts（账户）：存储卡号、余额、所属用户  
- Transactions（流水）：记录每一笔交易的始发账户、目标账户、金额及时间
